# Editing workflows

The graph is the main working surface. The brain map explains what the model is reacting to, but editing decisions should start from the response curve.

## Workflow A: compare 2-4 versions

Use this when you have several variants of the same creative.

![Compare workflow](assets/workflow-compare.svg)

1. Upload 2-4 versions of the same video idea.
2. Compare the overlaid curves.
3. Look for the upper-performing sections, not just the single winning file.
4. Build the next edit from the strongest blocks: best hook, best middle hold, best transition, best final useful section.
5. Run the combined edit again and compare it against the previous leader.

The goal is not to blindly choose one whole video. The goal is to use the strongest parts of each version as a practical editing map.

## Workflow B: improve one video

Use this when you only have one cut.

![Solo workflow](assets/workflow-solo.svg)

1. Open the predicted response graph.
2. Find real dips in the working part of the video.
3. Click the timestamp and inspect the nearby frames.
4. Compare the weak moment with stronger parts of the same video.
5. Test one edit at a time: trim, speed up, reframe, move text earlier, add a clearer action, or remove visual clutter.
6. Run the edited version again and check whether the graph improved.

The recommendation cards are hints. The curve is the control surface.
